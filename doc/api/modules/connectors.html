connectors Module

Defined in: src/connectors.js:413

Overview

Provides the connectors. A connector is responsible for connecting, communicating and determining the service status using the StatusResolver along with the appropriate ResolutionStrategy. Each connector handles a specific type of communication type with an endpoint. Specifically, the SocketConnector enables communication via raw socket (the data is transmitted as utf-8 encoded string) whereas the HttpConnector enables communication via http(s) protocol. An endpoint is actually an access point to the service. A service could have many access points, each of them is handled by one connector. Although the service status resolution is performed by the underline ResolutionStrategy, as mentioned above, there are cases where it is the connector itself that makes this decision. These cases can be summarized as follows:

For the HttpConnector

Http response status code status
404 down
500 down
302 undetermined
4xx (except of 404) undetermined
5xx (except of 500) undetermined
connection events error status
on error e.g. ECONNREFUSED, ENOTFOUND unreachable
on timeout no response is taken for specific period of time unreachable (if resolver's last attempt to determine the status fails)

For the SocketConnector

connection events error status
on error e.g. EHOSTUNREACH, ENOTFOUND unreachable
on timeout no response is taken for specific period of time unreachable (if resolver's last attempt to determine the status fails)
on end event remote host closes the connection undetermined (if resolver's last attempt to determine the status fails)

Configuration

The connector's configuration is described below (note that properties in [] are optional. when not set, the default values are used - those in parentheses.):

  • For the SocketConnector
    • [port] (9999), the service socket connection port.
    • [host] ('localhost'), the service host.
    • [timeout] (5000), the socket timeout, at ms.
    • [resolutionStrategy] (OnConnectionResolution), the applied resolution strategy.
  • For the HttpConnector
    • [url] (http://localhost:8080), the service endpoint url.
    • [timeout] (5000 ms), the http(s) request timeout, at ms.
    • [resolutionStrategy] (ResolutionStrategy), the applied resolution strategy.

API Usage samples

// socketConnectorFactory
var connectors = require('connectors');
var socketConnectorFactory = connectors.socketConnectorFactory;
var aResolutionStrategy = ...; //custom resolution strategy
var connector = socketConnectorFactory.create({
      host: '11.222.333.444',
      port: 55555,
      timeout: 2000,
      // if the property is not set, the default onConnectionResolution strategy is used.
      resolutionStrategy: aResolutionStrategy
  });
connector.addStatusResolvedListener(function onResolve(status) {
       logger.debug('Resolution callback is called');
 });
connector.start();

// httpConnectorFactory
var connectors = require('connectors').connectors;
var httpConnectorFactory = connectors.httpConnectorFactory;
var connector = httpConnectorFactory.create(({
     url: 'http://11.222.333.444:8080/',
     timeout: 3000
}));
connector.start();
